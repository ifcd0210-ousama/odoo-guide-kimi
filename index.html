Step 1

Create Your Odoo Docker Image

‚è± ~15 minutes

We'll create a custom Odoo 18.0 Community Docker image with a proper directory structure for addons and configuration. This approach keeps your custom code separate from the core Odoo installation.

1.2 Create the Dockerfile

üê≥ Dockerfile 
FROM odoo:18.0

USER root

# Install additional system dependencies
RUN apt-get update && apt-get install -y \
    build-essential \
    libxml2-dev \
    libxslt1-dev \
    libevent-dev \
    libsasl2-dev \
    libldap2-dev \
    libpq-dev \
    libpng-dev \
    libjpeg-dev \
    git \
    && rm -rf /var/lib/apt/lists/*

# Create directories for custom addons and data
RUN mkdir -p /mnt/extra-addons /var/lib/odoo/data /var/lib/odoo/sessions
RUN chown -R odoo:odoo /mnt/extra-addons /var/lib/odoo

# Copy custom configuration
COPY config/odoo.conf /etc/odoo/odoo.conf
RUN chown odoo:odoo /etc/odoo/odoo.conf

# Copy entrypoint script
COPY entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

USER odoo

# Expose Odoo services
EXPOSE 8069 8071 8072

ENTRYPOINT ["/entrypoint.sh"]
CMD ["odoo"]

1.4 Create Entrypoint Script

üöÄ entrypoint.sh 
#!/bin/bash

set -e

# Wait for PostgreSQL to be ready
if [ -n "$DB_HOST" ]; then
    echo "Waiting for PostgreSQL at $DB_HOST:$DB_PORT..."
    while ! nc -z $DB_HOST $DB_PORT; do
        sleep 0.1
    done
    echo "PostgreSQL started"
fi

# Set default values
: ${DB_HOST:=db}
: ${DB_PORT:=5432}
: ${DB_USER:=odoo}
: ${DB_PASSWORD:=odoo}
: ${DB_NAME:=postgres}

# Update odoo.conf with environment variables
if [ -f /etc/odoo/odoo.conf ]; then
    sed -i "s/db_host = .*/db_host = $DB_HOST/" /etc/odoo/odoo.conf || echo "db_host = $DB_HOST" >> /etc/odoo/odoo.conf
    sed -i "s/db_port = .*/db_port = $DB_PORT/" /etc/odoo/odoo.conf || echo "db_port = $DB_PORT" >> /etc/odoo/odoo.conf
    sed -i "s/db_user = .*/db_user = $DB_USER/" /etc/odoo/odoo.conf || echo "db_user = $DB_USER" >> /etc/odoo/odoo.conf
    sed -i "s/db_password = .*/db_password = $DB_PASSWORD/" /etc/odoo/odoo.conf || echo "db_password = $DB_PASSWORD" >> /etc/odoo/odoo.conf
fi

# Execute the original Odoo entrypoint
exec "$@"

1.5 Create Docker Compose

üìù docker-compose.yml 
version: '3.8'

services:
  db:
    image: postgres:16-alpine
    container_name: odoo-db
    restart: unless-stopped
    environment:
      POSTGRES_DB: postgres
      POSTGRES_USER: odoo
      POSTGRES_PASSWORD: odoo
      PGDATA: /var/lib/postgresql/data/pgdata
    volumes:
      - odoo-db-data:/var/lib/postgresql/data/pgdata
    networks:
      - odoo-network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U odoo"]
      interval: 5s
      timeout: 5s
      retries: 5

  odoo:
    build: .
    container_name: odoo-app
    restart: unless-stopped
    depends_on:
      db:
        condition: service_healthy
    environment:
      DB_HOST: db
      DB_PORT: 5432
      DB_USER: odoo
      DB_PASSWORD: odoo
      HOST: db
      USER: odoo
      PASSWORD: odoo
    volumes:
      - odoo-data:/var/lib/odoo
      - ./addons:/mnt/extra-addons:rw
    ports:
      - "8069:8069"
      - "8072:8072"
    networks:
      - odoo-network
    command: ["odoo", "--config", "/etc/odoo/odoo.conf"]

volumes:
  odoo-db-data:
  odoo-data:

networks:
  odoo-network:
    driver: bridge

Step 3

Expose with ngrok

‚è± ~10 minutes

ngrok creates secure tunnels to localhost, giving you a public URL for your local or self-hosted services. The free tier includes 1 static domain and up to 1GB/month data transfer.

3.1 Install ngrok

üì• Installation 
# Download ngrok (Linux x64)
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | \
  sudo gpg --dearmor -o /etc/apt/keyrings/ngrok.gpg && \
  echo "deb [signed-by=/etc/apt/keyrings/ngrok.gpg] https://ngrok-agent.s3.amazonaws.com buster main" | \
  sudo tee /etc/apt/sources.list.d/ngrok.list && \
  sudo apt update && sudo apt install ngrok

# Or use the standalone binary
wget https://bin.equinox.io/c/bNyj1mQVY4c/ngrok-v3-stable-linux-amd64.tgz
tar xvzf ngrok-v3-stable-linux-amd64.tgz
sudo mv ngrok /usr/local/bin/

3.2 Configure ngrok

üîë Authentication 
# Get your authtoken from https://dashboard.ngrok.com/get-started/your-authtoken
# Then configure ngrok:

ngrok config add-authtoken YOUR_AUTHTOKEN_HERE

# Verify configuration
ngrok config check

3.3 Start the Tunnel

üöÄ Start Tunnel 
# Basic tunnel to local Odoo
ngrok http 8069

# With a static domain (free tier includes 1 static domain)
ngrok http --domain=your-static-domain.ngrok-free.app 8069

# If Odoo is on a different server, use the IP
ngrok http http://your-server-ip:8069

# For HTTPS backend
ngrok http https://localhost:8069 --scheme http,https

3.4 Persistent ngrok with systemd

‚öôÔ∏è Create Systemd Service 
# Create systemd service file
sudo tee /etc/systemd/system/ngrok-odoo.service << 'EOF'
[Unit]
Description=ngrok tunnel for Odoo
After=network.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/ngrok http --domain=your-domain.ngrok-free.app 8069
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
EOF

# Reload systemd and enable service
sudo systemctl daemon-reload
sudo systemctl enable ngrok-odoo
sudo systemctl start ngrok-odoo

# Check status
sudo systemctl status ngrok-odoo

# View logs
sudo journalctl -u ngrok-odoo -f

3.5 ngrok Configuration File

üìÑ ~/.config/ngrok/ngrok.yml 
version: "3"
agent:
  authtoken: YOUR_AUTHTOKEN
  log_format: json
  log_level: info

üéÆ Using the Config

# Start all tunnels

# List active tunnels

Complete Setup Script

Here's a complete one-liner script that sets up everything on a fresh Ubuntu server:

üöÄ setup-odoo.sh 
#!/bin/bash
set -e

echo "üöÄ Odoo Community + Coolify + ngrok Setup Script"
echo "================================================"

# Update system
echo "üì¶ Updating system..."
sudo apt update && sudo apt upgrade -y

# Install Docker
echo "üê≥ Installing Docker..."
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# Install Docker Compose
echo "üìù Installing Docker Compose..."
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Install Coolify
echo "‚ùÑÔ∏è Installing Coolify..."
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash

# Install ngrok
echo "üåê Installing ngrok..."
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | \
  sudo gpg --dearmor -o /etc/apt/keyrings/ngrok.gpg && \
  echo "deb [signed-by=/etc/apt/keyrings/ngrok.gpg] https://ngrok-agent.s3.amazonaws.com buster main" | \
  sudo tee /etc/apt/sources.list.d/ngrok.list && \
  sudo apt update && sudo apt install ngrok -y

echo ""
echo "‚úÖ Installation Complete!"
echo ""
echo "Next steps:"
echo "1. Log out and log back in for Docker group changes"
echo "2. Configure ngrok: ngrok config add-authtoken YOUR_TOKEN"
echo "3. Access Coolify at http://$(hostname -I | awk '{print $1}'):8000"
echo "4. Clone your Odoo Docker project and deploy"
echo ""
echo "üéâ Happy deploying!"

sudo chown -R $USER:$USER ./addons
